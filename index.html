MODULE 02 — VULNERABILITY DISCOVERY & EXPLOITATION TOOLS

2. FINDING THE WEAKNESS

Systematically identify, validate, and exploit security vulnerabilities across the attack surface.

🎯 OWASP TOP 10 (2026)

The most critical web application security risks. Click each to learn detection techniques.

A01 — Broken Access Control▼
Detection: Test IDOR vulnerabilities by manipulating object references. Check for missing auth on admin endpoints. Try horizontal/vertical privilege escalation.

Tools: Burp Suite Repeater, Autorize extension, JWT_Tool

Commands:
curl -H "Cookie: session=admin" http://target.com/admin
A02 — Cryptographic Failures▼
Detection: Check for plaintext data transmission, weak encryption algorithms (MD5/SHA1), missing TLS, hardcoded secrets in source code.

Tools: SSL Labs, GitLeaks, TruffleHog
A03 — Injection (SQLi, XSS, Command)▼
Detection: Fuzz all input vectors with special characters. Test SQLi with ' OR '1'='1, time-based payloads. Test XSS with <script>alert(1)</script>.

Tools: SQLMap, XSSer, Commix, Burp Intruder
A05 — Security Misconfiguration▼
Detection: Check for default credentials, exposed .env files, directory listing enabled, verbose error messages, unnecessary features enabled.

Tools: Nikto, Nuclei, Dirb, Gobuster
